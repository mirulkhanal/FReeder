# Security Policy

## Supported Versions

The latest `main` branch is the supported version for security updates.

## Reporting a Vulnerability

Please do not open public issues for vulnerabilities. Report security concerns
privately to project maintainers first so fixes can be prepared responsibly.

Include:

- A clear description of the vulnerability.
- Steps to reproduce.
- Potential impact.
- Suggested remediation if available.

We will acknowledge receipt and provide status updates as triage progresses.
